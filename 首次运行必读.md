# ⚠️ 首次运行必读

## 🔴 看到 "Cannot find module 'express'" 错误？

这是**正常现象**，说明还没有安装依赖包。请按照下面步骤操作：

---

## ✅ 解决方法（选择一种）

### 方法1：让脚本自动安装（推荐）

#### Windows
1. **关闭当前窗口**
2. **重新双击** `启动服务器.bat`
3. 看到"首次运行，正在安装依赖..."时**耐心等待1-2分钟**
4. 不要关闭窗口，等待安装完成

#### Mac/Linux
```bash
bash 启动服务器.sh
```
首次运行会自动安装依赖，等待完成即可。

---

### 方法2：手动安装依赖

如果方法1失败，请手动安装：

#### Windows
1. 在当前文件夹按住 **Shift + 右键**
2. 选择 "在此处打开命令窗口" 或 "在此处打开PowerShell"
3. 输入并回车：
   ```
   npm install
   ```
4. 等待安装完成（1-2分钟）
5. 再次双击 `启动服务器.bat`

#### Mac/Linux
```bash
cd classroom-system-deploy
npm install
bash 启动服务器.sh
```

---

## 🔍 安装过程说明

安装依赖时你会看到：
```
npm WARN deprecated ...
npm notice created a lockfile as package-lock.json
added 57 packages from 40 contributors
```

这些都是**正常信息**，不是错误。

安装完成后会在文件夹里生成 `node_modules` 文件夹（约30MB）。

---

## 🚨 如果安装失败

### 错误1：网络连接问题
```
npm ERR! network request to https://registry.npmjs.org/express failed
```

**解决方法：**
1. 检查网络连接
2. 使用国内镜像：
   ```bash
   npm config set registry https://registry.npmmirror.com
   npm install
   ```

### 错误2：权限问题
```
npm ERR! code EACCES
```

**Windows解决方法：**
- 右键 `启动服务器.bat` → 以管理员身份运行

**Mac/Linux解决方法：**
```bash
sudo npm install
```

### 错误3：Node.js版本过低
```
npm ERR! engine Unsupported engine
```

**解决方法：**
- 访问 https://nodejs.org/
- 下载最新 LTS 版本（推荐 v18 或 v20）
- 安装后重试

## 🎯 检查清单

在运行前请确认：

- [ ] 已安装 Node.js（运行 `node --version` 检查）
- [ ] 文件已完全解压（不在压缩包里运行）
- [ ] 网络连接正常
- [ ] 杀毒软件没有阻止
- [ ] 有足够的磁盘空间（至少100MB）

---

## 📞 还是不行？

### 快速诊断
在命令行运行以下命令并截图：

```bash
node --version
npm --version
npm install
```

### 常见原因
1. **Node.js没装好** - 重新安装 Node.js
2. **网络问题** - 切换网络或使用国内镜像
3. **防火墙/杀毒软件阻止** - 临时关闭后重试
4. **文件损坏** - 重新下载压缩包
